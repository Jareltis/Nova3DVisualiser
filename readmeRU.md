# Nova3DVisualiser

3D-движок реального времени, который рендерит изображение в терминал в виде ASCII-арта, написанный на C# / .NET 10. Это рейтрейсер — по умолчанию на CPU, с опциональным путём на GPU (ILGPU / CUDA) при полном паритете: каждый кадр он трассирует лучи по сцене, отображает яркость в градиент символов и (на терминалах с truecolor) подкрашивает каждую ячейку 24-битным RGB-цветом, рисуя результат прямо в консоли.

В комплекте идёт мультиплеерное демо-приложение с полноценным **встроенным редактором сцены** (с переработанным HUD, управляемым мышью и клавиатурой), **системой миров** (каждая сцена — это один JSON-файл, который можно создать, сохранить и загрузить заново), **богатым цветным освещением**, **PNG-текстурами**, слоем **настоящей физики твёрдых тел** (ходячий персонаж от первого лица плюс правдоподобная гравитация — угол падения, качение, отскок, трение, масса и 3D-вращение), **системой камер с несколькими видами** и базовой TCP-сетью с чатом. Весь его UI — включая мастер настройки — работает на собственном консольном рендерере движка (без внешнего UI-тулкита).

## Возможности

### Рендеринг
- **Рендерер CPU или GPU, выбирается для каждого мира.** По умолчанию — многопоточный трассировщик на CPU. Мир можно переключить на **GPU** (NVIDIA, через [ILGPU](https://ilgpu.net/) → CUDA; также работает на любом OpenCL-устройстве): все первичные, теневые и осветительные лучи считаются на видеокарте. GPU-путь обеспечивает **полный паритет фич** с CPU: front-to-back прозрачность, все виды света (point / directional / spot / area), веера лучей spot, формы конуса circle/square/triangle и мягкие тени area — совпадение проверяется попиксельно self-тестом `gputest`. Используется такое же **двухуровневое BVH-ускорение**, как на CPU (BVH каждого меша в локальном пространстве строится и грузится один раз; ядро переводит луч в пространство объекта и обходит уплощённое дерево без стека), поэтому тяжёлые меши остаются быстрыми. Если мир запросил GPU, но пригодной видеокарты нет, происходит прозрачный откат на CPU. Выбирается в диалоге **Создать мир** (**Renderer: CPU / GPU**).
- Трассировка лучей на CPU с выводом в ASCII через градиент яркости.
- **Truecolor-вывод**: каждая ячейка несёт 24-битный RGB-цвет (`Rgb24`), выводимый ANSI-кодами, поэтому картинка больше не ограничена 16 цветами консоли. На терминалах без truecolor используется ближайший консольный цвет.
- **Два вида прозрачности**: у каждого объекта есть и **прозрачность объекта** — **альфа**-канал цвета, делающий его сквозным (перекрывающиеся прозрачные поверхности компонуются **front-to-back** через depth peeling, а прозрачный объект отбрасывает более светлую тень) — и **блёклость цвета** (значение **Pale**): отдельная ручка, которая выцветает собственный цвет поверхности к белому, при этом объект остаётся полностью твёрдым и даёт полную тень.
- **PNG-текстуры + UV**: любой объект может носить **PNG**-картинку (8-бит RGB / RGBA), декодируемую внутри приложения (без сторонней библиотеки изображений). Текстуры мапятся по UV на каждый угол — генерируются процедурно для примитивов (cube / sphere / cylinder / cone / pyramid / ramp / flatpicture; сфера использует **эквиректангулярную** карту) и читаются из `.obj` для импортированных мешей. Для каждого объекта можно задать **тайлинг** (`TexScale`), **какая грань** её носит (`TexFace` — например, одна из шести сторон куба) и **фильтрацию** — **nearest**, **bilinear** или **mipmapped** (трилинейная минификация, чтобы дальние / скользящие поверхности не мерцали). Текстуры рендерятся **одинаково на CPU и GPU** (проверяется `gputest`), а в мультиплеере **байты PNG передаются пирам**, поэтому клиент без файла всё равно видит текстуру.
- Конвейер освещения с плавающей точкой, поканальной **тональной компрессией по Рейнхарду (Reinhard)**, фоновой (ambient) составляющей и регулируемой экспозицией.
- Сглаженное (по нормалям вершин) затенение, с дружелюбным к Blender загрузчиком OBJ (триангуляция n-угольников веером, отрицательные/относительные индексы, запасной вариант с геометрической нормалью).
- **BVH-ускорение** для высокополигональных мешей, строится один раз в локальном пространстве и обходится с лучом, преобразуемым каждый кадр, поэтому работает даже с анимированными/вращающимися моделями.
- Отсечение по ограничивающей сфере для каждого объекта — как для первичных лучей, так и для теневых.
- Опциональные тени (переключаются для каждого мира).
- Регулируемая детализация рендера (1–4): меняйте разрешение на скорость прямо в процессе одной клавишей.
- **Diff-рендеринг**: каждый кадр в консоль перезаписываются только реально изменившиеся ячейки (с прыжками курсора по ANSI), а не весь экран целиком. При неподвижной камере выводится почти ничего — это убирает «прорисовку сверху вниз» (тиринг) и повышает частоту кадров.

### Освещение
В Nova реализована полноценная система множественного освещения. Каждый источник света проверяется на тень **независимо** и вносит вклад аддитивно, поэтому поверхность, затенённая от одного источника, всё равно освещается другим (нет единственной доминирующей тени). Тени **альфа-корректные**: непрозрачный объект отбрасывает полную тень, а прозрачный — соответственно более светлую (теневой луч накапливает альфу каждого прозрачного окклюдера). Виды источников:

- **Point (точечный)** — всенаправленный, с обратно-квадратичным затуханием.
- **Directional (направленный)** — параллельное «солнце» с постоянным мягким затуханием.
- **Spot (прожектор)** — конус, направленный вдоль вектора, с дополнительным формированием (см. ниже).
- **Area (площадной)** — квадратный излучатель, сэмплируемый по сетке, для площадного затухания и мягких теней.

Дополнительно для каждого источника:

- **Цветное освещение** — каждый источник излучает RGB-цвет (берётся из его `ConsoleColor`). Параметр `SurfaceTint` позволяет цветному свету проявляться на любой поверхности, а не только на поверхности того же цвета.
- **Spin (вращение)** — со временем поворачивает направление источника (рад/с) для анимированного освещения.
- **Многонаправленные лучи прожектора** — прожектор может бросать сразу **1–8 конусов** (`Beams`), равномерно развёрнутых вокруг его оси; точка освещается тем лучом, который до неё дотягивается.
- **Форма конуса прожектора** — сечение конуса может быть **кругом (Circle)**, **квадратом (Square)** или **треугольником (Triangle)**.

### Физика (опционально, для каждого мира)
В Nova реализован **настоящий движок физики твёрдых тел** — единый **решатель на импульсах со связями** (последовательные импульсы / PGS по манифолду контактов, с прогревом, трением, трением качения, упругостью, позиционной коррекцией split-impulse и засыпанием островов). Даёт правдоподобную гравитацию с углом падения, инерцией, качением, отскоком, вращением, **устойчивыми стопками и естественным кувырканием** — а не скриптовое падение. Два главных переключателя уровня мира, оба задаются при создании мира (и редактируются в JSON):

- **Коллизия (Collision)** *(по умолчанию включена)* — главный выключатель твёрдых объектов. **Игрока** (пузырь камеры) выталкивает из пола и любого коллайдера, так что сквозь них нельзя пройти или пролететь. Каждый меш может использовать быстрый **AABB** (бокс по мировым осям) или **OBB** — ориентированный бокс, плотно облегающий наклонённую форму (контакты бокс-бокс считаются полным 3D-тестом разделяющих осей). У каждого объекта есть свой флаг **Collide**; когда переключатель мира **выключен**, Collide у всех объектов принудительно выключается и блокируется.
- **Гравитация (Gravity)** *(по умолчанию выключена)* — включает симуляцию:
  - **Игрок становится ходячим персонажем**: гравитация тянет камеру вниз, она стоит на поверхностях, **Space** — прыжок, **F1** переключает режим свободного полёта для постройки.
  - Любой объект может включить **Gravity** — меши, примитивы, сферы, даже свет или платформа падают и ложатся на то, что под ними.
  - Падающий **шар сталкивается с реальной поверхностью**, на которую падает — с настоящими треугольниками пирамиды / рампы / меша, **а не с габаритным боксом** — поэтому он **скатывается по склону с настоящим импульсом** (быстрее на крутом), доезжает до остановки на ровном и **отскакивает** по своей упругости. Падающий меш так же ложится на реальную поверхность под ним.
  - **Ящики лежат, скользят или опрокидываются на реальной поверхности — их не выбрасывает.** Ящик, брошенный на рампу, ложится **плашмя на склон** и съезжает или скатывается по нему; его **не выкидывает** вбок (ящик сталкивается с настоящими треугольниками меша, а не с габаритным боксом). **Скользит он или кувыркается** — зависит от **крутизны склона и трения**: на пологом склоне просто скользит, на крутой грани / будучи неустойчивым — **переваливается через грань** и катится с грани на грань, успокаиваясь внизу — всё **естественно** из решателя.
  - **Всё сталкивается со всем, и ящики СТОПКУЮТСЯ.** Ящики и шары сталкиваются в любых сочетаниях (ящик-ящик, шар-ящик, шар-шар), с учётом массы, так что **башня из ящиков стоит устойчиво** (без дрожи, крена и проваливания), а **шар, влетевший в неё, разбрасывает башню** — и затем вся куча **успокаивается и засыпает** вместе.
  - **Трение качения** — катящийся шар и кувыркающийся ящик **замедляются и останавливаются**, а не катятся вечно, при этом всё ещё скатываясь **вниз** по склону (гравитация сильнее малого сопротивления качению). Настраивается на каждый объект.
  - **Упругость (Bounce / restitution)** задаётся на весь мир (диалог создания) и на каждый объект: `0` = мёртвая остановка, `1` = идеально упруго. На контакте упругости двух поверхностей **комбинируются**, поэтому прыгучий шар на «мёртвом» полу ведёт себя иначе, чем на стене-«батуте».
  - **Трение, масса и 3D-вращение**: объекты теряют скорость на трении (Кулона + качения), тяжёлый объект сдвигается меньше (пер-объектная **Mass**), а удар не по центру заставляет тело **кувыркаться** — полное 3D-вращение с настоящим тензором инерции и кватернионной ориентацией без дрейфа.
  - Симуляция дробится на **под-шаги** внутри, поэтому ведёт себя одинаково, рендерит ли терминал на 60 FPS или на нескольких, и имеет предохранители, чтобы ничего не улетало.

Физика объектов считается **у authority/solo**, а полное состояние каждого сдвинутого тела — позиция, ориентация и скорость — транслируется клиентам, которые **экстраполируют и плавно подводят** к нему, поэтому падение, качение и кувыркание объектов синхронно видят все (а не «замороженными» между обновлениями); гравитация самого игрока считается локально у каждого участника, поэтому ничего не рассинхронизируется.

### Система миров и редактор
- **Мир** — это один файл `worlds/<name>.json`, владеющий всей сценой: переключатели графики, платформа и полный трансформ каждого объекта (позиция/поворот/масштаб/цвет/привязка, а для источников света — вид/направление/конус/лучи/форма и т.д.).
- `models/` — это **чистая библиотека мешей**, только файлы `<name>.obj`. Мир ссылается на меш по имени и задаёт его размещение.
- **Встроенный редактор сцены** (включается клавишей `Tab`) позволяет создавать, перемещать, поворачивать, масштабировать, перекрашивать и удалять объекты вживую, а затем сохранять обратно в JSON.
- Встроенные типы для спавна: `cube`, `sphere`, `cylinder`, `cone`, `pyramid`, `ramp` (клин с наклонной гранью — удобно для скатывания), `flatpicture` (двусторонняя вертикальная панель с текстурой — постер/билборд, без коллизии), `light`, плюс каждый меш из `models/`.
- **Формы платформы**: квадрат, прямоугольник или круг, с настраиваемым размером/цветом.
- **Живые тумблеры графики**: переключайте **тени** (`F2`), **BVH-ускорение** (`F3`), **свет камеры** (`F4`) и **пол-платформу** (`F6`) прямо в сцене — без пересоздания мира. На сервере эти изменения синхронизируются всем клиентам.

### Камеры, виды и HUD
- **Вы — это тело + камера.** От первого лица камера на уровне глаз; **F7** переключает на **3-е лицо** (сзади и выше) и **2-е лицо** (спереди, смотрит назад на вас — видно своё лицо). Внешние виды подтягивают камеру внутрь, если на пути стена; другие игроки в любом случае видят ваш аватар.
- **Размещаемые камеры.** Заспавните объект **camera** (точку обзора): **Fixed** (неподвижная) — статичный кадр с места установки; **Follow** (следящая) — поворачивается к цели (по умолчанию вашему телу, либо любому объекту по id). **F8** переключает активный вид между вашим телом и каждой размещённой камерой — локальный выбор каждого игрока, так что клиенты тоже могут смотреть через камеры.
- **Разделённый экран.** **F9** делит экран на 2 части — ваш активный вид слева, следующий вид в цикле F8 справа — две камеры рендерятся одновременно.
- **Три режима HUD.** **PLAY** (минимальный прицел поверх полноэкранного 3D), **overlay-edit** (**Tab** — панели редактора поверх 3D) и **docked-edit** (**`** — компоновка в стиле Unity/Blender: 3D в центральном вьюпорте, панель инструментов сверху, строка состояния снизу, навигируемая **иерархия** слева и сгруппированный **инспектор** справа). Чат — обособлённое окно с переносом строк и прокруткой, которое никогда не наезжает на HUD. Весь HUD **перестраивается при изменении размера или масштаба (зуме) терминала**.

### Сеть
- Базовый TCP сервер/клиент с общей сценой и встроенным чатом. К одному серверу могут подключаться **несколько клиентов**, и каждый игрок видит остальных как небольшой подвижный **аватар**.
- **Сервер — это владелец мира (authority)**: он отправляет полный мир подключающимся клиентам — большие меши режутся на **чанки** для надёжной передачи — а затем транслирует дельты правок, изменения настроек графики и физику (позиции **и вращение** объектов). Клиенты могут летать и осматривать сцену, но изменять общий мир может только authority.

### Прочее
- Логирование в файл с отметками времени.
- Контроль частоты кадров (ограничение FPS), дельта-время высокого разрешения и клавиша выхода.

## Требования
- .NET 10 SDK.
- Windows (ввод с клавиатуры использует Win32 API, а поддержка мыши в мастере — Win32 console API, поэтому приложение сейчас работает только под Windows).
- Терминал с поддержкой 24-битного ANSI-цвета для наилучшего вывода (рекомендуется Windows Terminal).

## Сборка и запуск
```bash
dotnet build -c Release
```
Установите `SampleGame` как стартовый проект в Visual Studio или запустите его напрямую:
```bash
dotnet run --project SampleGame -c Release
```

### Настройка при запуске (меню перед рендером)
Перед циклом рендеринга запускается брендированный мастер настройки. Он рисуется **собственным консольным UI движка** (без внешнего UI-тулкита) и управляется **клавиатурой и мышью**; он перестраивается при изменении размера терминала:
1. **Режим сессии** — локальная одиночная или онлайн.
2. **Сетевая роль** (только онлайн) — поднять **Server** или подключиться как **Client**.
3. **Мир** — **Создать** новый мир или **Загрузить** сохранённый (`worlds/*.json`).
   - **Создание** позволяет задать имя мира, переключатели (**Shadows** — тени, **BVH acceleration** — BVH-ускорение, **Extra fixed light** — дополнительный фиксированный свет, **Disable camera light** — отключить свет камеры, **Include platform** — включить платформу), **Renderer** (рендерер: **CPU** / **GPU (NVIDIA)**), **форму платформы** (Square / Rectangle / Circle) и её размер/ширину/глубину, а также переключатели **физики** (**Gravity** — гравитация и её сила + **Bounce** — упругость отскока 0–1, **Collision** — коллизия).
4. **Сеть** (только онлайн) — порт прослушивания (сервер) или IP сервера + порт (клиент).

Клиент, подключившийся к серверу, скачивает мир хоста автоматически.

> **Зум:** мастер перестраивается при **изменении размера окна** или при **клавиатурном зуме шрифта** (Ctrl+= / Ctrl+-). Зум по **Ctrl+колёсико** не меняет размер мастера (он перехватывает колёсико мыши для кликов) — внутриигровое 3D по-прежнему реагирует на Ctrl+scroll.

## Управление

Клавиши сгруппированы по режимам. Управление камерой (полёт) работает всегда; управление редактором действует только при открытом редакторе (`Tab`); чат перехватывает весь ввод, пока он открыт (`T`).

### Камера / полёт (активно всегда)
| Клавиша | Действие |
| --- | --- |
| **W** / **S** | Движение вперёд / назад (относительно направления взгляда) |
| **A** / **D** | Боковое смещение влево / вправо |
| **Space** / **C** | Полёт вверх / вниз *(режим полёта)* — в **мире с гравитацией** **Space** прыгает, а **C** не используется |
| **F1** | Переключить **полёт** ↔ **ходьбу**. В мире с гравитацией по умолчанию — **ходьба** (гравитация тянет вниз, вы стоите на полу/объектах и не проходите сквозь них); режим полёта возвращает свободное движение вверх/вниз и проход сквозь объекты |
| **F7** | Циклически переключать вид **от 1-го → от 3-го → от 2-го лица**. Вы — это **тело + камера**: от 1-го лица камера на уровне глаз (своё тело скрыто); от 3-го лица она отходит назад и вверх; от 2-го лица она сидит **спереди** и смотрит **назад** на вас (видно своё лицо). И 3-е, и 2-е лицо подтягивают камеру внутрь, если на пути стена. Другие игроки в любом случае видят ваш аватар |
| **F8** | Циклически переключать **активный вид** между **видом от тела** и каждым размещённым **объектом-камерой**: **Fixed** (неподвижная) камера показывает статичный кадр с места установки; **Follow** (следящая) камера поворачивается к **цели** — по умолчанию вашему телу, либо любому объекту по id (поле **Target**). Управление телом при просмотре с камеры сохраняется (ваш аватар показывается, чтобы вы видели себя). Это локальный выбор каждого игрока — клиенты тоже могут смотреть через камеры |
| **F9** | Переключить **разделённый экран**: ОДИН (весь экран) ↔ **2 части СЛЕВА \| СПРАВА**. Левая половина — ваш текущий активный вид (F8); правая — **следующий** вид в цикле F8 (следующая размещённая камера, или вид от тела, если камер нет) — две камеры рендерятся одновременно. Локальный выбор каждого игрока |
| **← / →** | Поворот взгляда влево / вправо (рыскание) |
| **↑ / ↓** | Поворот взгляда вверх / вниз (тангаж) |
| **Q** / **E** | Крен камеры влево / вправо |
| **R** | Сбросить крен в горизонт |
| **Z** / **X** | Приблизить / отдалить (FOV уже / шире, 20°–120°) |
| **V** | Сбросить FOV к значению по умолчанию |
| **=** / **-** | Увеличить / уменьшить мощность света камеры |
| **P** | Циклически менять детализацию рендера 1 → 2 → 3 → 4 → 1 (меньше = быстрее) |
| **T** | Открыть чат — обособлённое окно (не перекрывает HUD) с переносом длинных сообщений; **PgUp/PgDn** или **↑/↓** прокручивают историю; **Enter** отправляет, **Esc** закрывает чат |
| **Esc** | Выйти из приложения (во время чата Esc закрывает чат) |

> **Гравитация** — настройка мира (включается в диалоге *Создать мир*). Если гравитация **выключена**, камера всегда свободно летает, как раньше. Если **включена**, вы начинаете как персонаж, идущий по земле — нажмите **F1**, чтобы в любой момент переключиться в режим полёта для постройки/осмотра.

### Редактор (**Tab** — оверлейный редактор, **`** — стыкованный редактор)
У HUD три режима: **PLAY** (минимум — прицел + подсказка чата поверх полноэкранного 3D), **OVERLAY-EDIT** (**Tab** — панели редактора поверх полноэкранного 3D) и **DOCKED-EDIT** (**`** — компоновка в стиле Unity/Blender: 3D в центральном вьюпорте, панель инструментов сверху, строка состояния снизу, иерархия слева и сгруппированный инспектор справа). Управление редактированием ниже одинаково в обоих режимах — отличается только оформление. Управление полётом работает во время редактирования. Компоновка перестраивается при изменении размера/масштаба терминала.

| Клавиша | Действие |
| --- | --- |
| **Tab** | Оверлейный редактор (Play ↔ Overlay-Edit) |
| **`** | Стыкованный редактор (Play ↔ Docked-Edit) |
| **G** | Переключить тип для спавна (cube / sphere / cylinder / cone / pyramid / ramp / flatpicture / light / **camera** / любой меш из `models/`). **camera** — размещаемая точка обзора (задайте её **Kind**: Fixed или Follow; у Follow-камеры поле **Target** — id объекта для слежения, по умолчанию игрок), на которую переключаются клавишей **F8**. Вращение камеры по конвенции обзора: **Rot X = крен (roll)** (крутит картинку), **Rot Y = рыскание (yaw)**, **Rot Z = тангаж (pitch)** — наводите Y/Z |
| **B** | Заспавнить текущий тип перед камерой *(только authority)* |
| **F** | Выбор прицелом: взять объект под прицелом |
| **[** / **]** | Выбрать предыдущий / следующий объект |
| **,** / **.** | Перемещать курсор поля вверх / вниз в панели свойств |
| **N** / **M** | Уменьшить / увеличить значение выделенного поля *(только authority)* |
| **Enter** | Ввести точное значение выделенного поля — **Name** (текст) или любое **числовое** поле (при Enter значение парсится и ограничивается допустимым диапазоном; Esc отменяет). Поля-переключатели вводом не задаются — используйте N/M *(только authority)* |
| **I** / **K** | Двигать выбранный объект по +Z / −Z *(только authority)* |
| **J** / **L** | Двигать выбранный объект по −X / +X *(только authority)* |
| **U** / **O** | Двигать выбранный объект вверх / вниз (+Y / −Y) *(только authority)* |
| **Delete** | Удалить выбранный объект *(только authority)* |
| **F2** | Включить/выключить тени *(только authority, синхронизируется вживую)* |
| **F3** | Включить/выключить BVH-ускорение *(только authority, синхронизируется вживую)* |
| **F4** | Включить/выключить свет камеры *(только authority, синхронизируется вживую)* |
| **F6** | Включить/выключить пол-платформу *(только authority, синхронизируется вживую)* |
| **F5** | Сохранить мир обратно в `worlds/<name>.json` *(только authority)* |

> В онлайн-сессии спавнить/двигать/редактировать/удалять/сохранять может только **authority** (сервер или локальная одиночная сессия). Подключённый клиент может открыть редактор, чтобы **выбирать и осматривать** объекты, но его изменения не затрагивают общий мир.

| --- | --- |
| *ввод* | Набрать сообщение |

### Панель свойств (редактируется клавишами `,` `.` `N` `M` или вводом по **Enter**)
Клавишами `,` / `.` двигайте курсор поля, а **N** / **M** — шагают значение выделенного поля. Для быстрого и точного изменения нажмите **Enter**, чтобы *ввести* значение выделенного поля: **Name** (текст) или любое **числовое** поле (Pos/Rot/Scale/Spin, R/G/B/A, Pale, Mass, Bounce, Friction, RollFric, TexScale, Radius, Power и поля источника света). Введённое значение при **Enter** парсится и **ограничивается допустимым диапазоном** поля; **Esc** отменяет (оставляет без изменений); пустой или неверный ввод игнорируется. Поля-переключатели (Collide, Gravity, Collider, Texture, TexFace, TexFilter, Kind, Shape) вводом *не* задаются — листайте их клавишами N/M.

Список редактируемых полей зависит от типа выбранного объекта:

- **Меш / cube / cylinder / cone / pyramid / ramp** — Pos X/Y/Z, Rot X/Y/Z, Scale (масштаб), Spin (скорость авто-вращения), **R / G / B / A** (цвет + прозрачность объекта), **Pale** (блёклость цвета), **Texture** (листает PNG из `textures/` + «none»), **TexScale** (тайлинг UV), **TexFace** (какая грань носит текстуру — 6 граней куба, иначе «All»), **Collide** (коллизия), **Gravity** (гравитация), **Collider** (форма коллайдера), **Mass** (масса), **Bounce** (упругость).
- **Flat picture** — Pos X/Y/Z, Rot X/Y/Z, Scale, **R / G / B / A**, **Pale**, **Texture**, **TexScale**, **TexFace** — визуальная панель (без физических полей; не коллайдит).
- **Sphere** — Pos X/Y/Z, Radius (радиус), **R / G / B / A**, **Pale**, **Texture**, **TexScale**, **TexFace**, **Collide**, **Gravity**, **Mass**, **Bounce**.
- **Платформа** — Pos X/Y/Z, Shape/Size (или Width × Depth), **R / G / B / A**, **Pale**, **Collide**, **Gravity**.
- **Light (источник света)** — Pos X/Y/Z, Power (мощность), **Influence** (влияние цвета), **R / G / B / A**, **Pale**, **Kind** (вид), затем в зависимости от вида, затем **Gravity**:
  - *Point*: ничего дополнительно.
  - *Directional*: Dir X/Y/Z (направление), Spin.
  - *Spot*: Dir X/Y/Z, Cone (половинный угол), **Beams** (1–8 развёрнутых конусов), **Shape** (Circle / Square / Triangle), Spin.
  - *Area*: Dir X/Y/Z, Size (половинный размер), Spin.

Цвет редактируется четырьмя каналами **R / G / B / A** (0–255 каждый): полный 24-битный цвет плюс канал **A** (альфа) — это **прозрачность объекта** (насколько он сквозной). Отдельное значение **Pale** (0–1) — это **прозрачность цвета**: выцветает собственный цвет поверхности к белому, НЕ влияя на сквозность и тень. Для источника света R/G/B задают излучаемый цвет, **Pale** выцветает эту эмиссию к белому, а **Influence** — насколько сильно цвет подкрашивает поверхности. **Collide** / **Gravity** — переключатели вкл/выкл (N или M меняют их); они показывают `Off (locked)` и не могут быть включены, пока в мире выключен соответствующий переключатель Коллизии / Гравитации. **Collider** переключает форму коллайдера меша между **AABB** (быстрый бокс по мировым осям) и **OBB** (ориентированный бокс, поворачивается вместе с объектом и плотнее облегает его форму — лучше для наклонённых/вытянутых мешей). **Mass** (масса) задаёт вес в импульсном солвере физики — более тяжёлый объект меньше сдвигается при столкновении с более лёгким. **Bounce** — упругость отскока объекта (`0` = «мёртвый», `1` = идеально упругий); показывает `world (X)`, когда наследует упругость мира, а на контакте упругости двух поверхностей комбинируются.

## Добавление моделей
Положите меш в `SampleGame/models/` как `<name>.obj` (экспортируйте с нормалями вершин для сглаженного затенения; Blender делает это по умолчанию). Модели — это чистая библиотека геометрии, **отдельного JSON на модель больше нет**; размещение (позиция, поворот, масштаб, цвет, привязка, вращение) хранится в мире.

Чтобы разместить модель в сцене, либо:
- Заспавните её вживую в редакторе (`G`, чтобы найти её в списке спавна, затем `B`, чтобы заспавнить), расставьте и нажмите `F5`, чтобы сохранить; либо
- Добавьте запись объекта в `worlds/<name>.json` вручную.

Объект мира выглядит так:
```json
{
  "id": 0,
  "type": "mesh",
  "mesh": "monkey",
  "position": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "rotation": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "scale": 1.0,
  "color": "Yellow",
  "anchor": "bottom",
  "rotateSpeed": 1.0,
  "collides": true,
  "gravity": false
}
```
- `type` — `mesh`, `cube`, `sphere`, `cylinder`, `cone`, `pyramid`, `ramp`, `flatpicture` или `light`.
- `mesh` — имя файла в `models/` (без `.obj`), для `type: "mesh"`.
- `texture` — имя PNG-файла из `textures/` (напр. `"brick.png"`, `""` = нет/плоский цвет): накладывает картинку на объект. Работает на всех примитивах (cube/sphere/cylinder/cone/pyramid/ramp/flatpicture) **и на импортированных `mesh`-объектах** (мапятся по UV, заданным в `.obj`). В онлайне **байты PNG передаются пирам**, поэтому клиент без файла всё равно видит текстуру (синхронизируется именем файла *и* пикселями).
- `textureScale` — коэффициент тайлинга UV (деф `1`): `2` тайлит текстуру 2×2, `0.5` показывает половину и т.д.
- `textureFace` — какая грань носит текстуру (деф `-1` = все грани). Для куба `0`–`5` выбирают одну сторону (`+X, -X, +Y, -Y, +Z, -Z`), остальные тогда плоского цвета. У других форм одна грань (применимо только `-1`/все).
- `textureFilter` — сэмплирование текстуры: `0` = **nearest** (по умолчанию), `1` = **bilinear** (сглаживает увеличение), `2` = **mipmapped** (трилинейная минификация, чтобы дальние / скользящие поверхности не мерцали).
- `position` — позиция в мире в единицах (Y — вертикальная ось).
- `rotation` — начальный поворот в радианах.
- `scale` — равномерный масштаб.
- `color` — hex-строка `#RRGGBB` (или `#RRGGBBAA`, где альфа = прозрачность ОБЪЕКТА), тройка `R,G,B` / `R,G,B,A`, либо имя .NET `ConsoleColor` (например, `Red`). Сохраняется обратно в hex.
- `colorFade` — прозрачность ЦВЕТА / блёклость, 0–1 (по умолчанию `0`): выцветает цвет объекта к белому, независимо от альфы (`A` = прозрачность объекта, `colorFade` = прозрачность цвета).
- `anchor` — как меш располагается в своей позиции: `bottom` (основание на полу, по умолчанию), `center` (геометрический центр) или `origin` (исходное начало координат OBJ).
- `rotateSpeed` — автоматическое вращение вокруг вертикальной оси, в радианах в секунду (0 = статично).
- `collides` — является ли объект твёрдым (действует, только когда в мире `physics.collisionEnabled` = true). По умолчанию `true`.
- `gravity` — притягивается ли объект гравитацией мира вниз (действует, только когда `physics.gravityEnabled` = true). По умолчанию `false`.
- `collider` — форма коллайдера меша: `"aabb"` (бокс по мировым осям, по умолчанию) или `"obb"` (ориентированный бокс, поворачивается вместе с объектом).
- `mass` — масса в импульсном солвере (по умолчанию `1`); тяжёлый объект меньше сдвигается при столкновении.
- `restitution` — упругость 0–1; `-1` (по умолчанию) наследует `physics.restitution` мира.

Объект `light` дополнительно использует: `power`, `lightKind` (`point`/`directional`/`spot`/`area`), `direction`, `coneAngle`, `beamCount`, `coneShape` (`circle`/`square`/`triangle`), `lightSize` и `lightSpin`.

В файле мира также есть верхнеуровневый блок `physics` с главными переключателями:
```json
"physics": { "gravityEnabled": false, "gravityStrength": 9.8, "collisionEnabled": true, "restitution": 0.0 }
```
(`restitution` — упругость мира по умолчанию, 0–1; отдельные объекты могут её переопределить.)

Отредактируйте модель или JSON мира и просто перезапустите — пересборка не требуется.

## Структура проекта
- `Nova3DVisualiser/` — движок (библиотека): векторная математика, фигуры, трассировка лучей, конвейер цветного освещения, truecolor-вывод в консоль, BVH, загрузка OBJ/мешей, сеть, логирование. Не зависит от сторонних пакетов.
- `Nova3DVisualiser.Gpu/` — опциональный GPU-рендерер (зависит от движка + [ILGPU](https://ilgpu.net/)). Держит тяжёлую GPU-зависимость вне движка; содержит `GpuScreen`, ядро трассировки NVIDIA/CUDA и загрузку плоского снимка сцены.
- `SampleGame/` — демонстрационное приложение: мастер настройки (свой небольшой консольный UI-тулкит в `SampleGame/WizardUi/` — клавиатура + мышь, без внешней UI-библиотеки), система миров (`worlds/`), встроенный редактор сцены, сетевой хост и библиотека мешей `models/`.

## Участие
Nova3DVisualiser пишется исключительно его мейнтейнером и **не принимает вклад в код** — пул-реквесты закрываются без слияния, а форки обратно не вливаются. При этом **отчёты об ошибках, предложения функций и идеи очень приветствуются** через [GitHub Issues](https://github.com/Jareltis/Nova3DVisualiser/issues). Основные изменения по сравнению с оригиналом: переименование движка; переписанный конвейер освещения с плавающей точкой и **цветом**, поканальной тональной компрессией и **24-битным truecolor**-выводом; полноценная система множественного освещения (point / directional / spot / area, цветное излучение, вращение света, многолучевые прожекторы и формы конуса круг/квадрат/треугольник); сглаженное затенение и устойчивый к Blender загрузчик OBJ; **система миров** на JSON со **встроенным редактором сцены** и сетевой синхронизацией вживую; **опциональный GPU-рендерер** (ILGPU / CUDA) при полном паритете с CPU; **PNG-текстуры** (UV-маппинг, bilinear + mipmap фильтрация, сетевая синхронизация пикселей); **система камер с несколькими видами** (1-е / 2-е / 3-е лицо, размещаемые Fixed / Follow камеры и разделённый экран); переработанный трёхрежимный **HUD** (play / overlay-edit / стыкованный иерархия + инспектор), причём весь UI — включая мастер настройки — на собственном консольном рендерере движка (без внешнего UI-тулкита); слой **настоящей физики твёрдых тел** (ходячий персонаж от первого лица; гравитация, где шар сталкивается с реальной поверхностью меша и скатывается по склонам с импульсом; коллизия по объектам с AABB/OBB-коллайдерами и полным 3D-тестом разделяющих осей; пер-объектные упругость, масса, трение и 3D-кувыркание с кватернионной ориентацией без дрейфа; кадронезависимый substep; всё под главными переключателями уровня мира); **diff-рендеринг** консоли; загрузка мешей из папки с привязкой (anchor); BVH-ускорение; отсечение по ограничивающей сфере для первичных и теневых лучей; переключатель теней; логирование с отметками времени; и улучшения главного цикла (клавиша выхода, ограничение FPS, дельта-время высокого разрешения).
