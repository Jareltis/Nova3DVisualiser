## О проекте

Neo 3D — минималистичный трёхмерный движок на C#, работающий полностью внутри системной консоли. Проект построен без использования сторонних графических библиотек. Все вычисления — от кастомной векторной математики до трассировки лучей (Raycasting/Raytracing) и сетевого кода — выполняются на процессоре (CPU). В будущем планируется перенос самых тяжёлых вычислений с процессора на видеокарту с помощью Vulkan.

Проект создан в демонстрационных и образовательных целях для [YouTube-видео](https://youtube.com/ссылка_на_видео).

**Ветка по умолчанию содержит стабильную версию проекта. Также есть долгоживущая ветка development и кратковременные ветки с фичами.**

## Возможности

- Отрисовка геометрической сферы и `.obj`-файла путём трассировки лучей (Raytracing).
- Оптимизация пересечений через предварительную проверку попадания в ограничивающую сферу (Bounding Sphere) перед детальным просчётом полигонов.
- Расчёт затухания света в зависимости от расстояния.
- Диффузное освещение Ламберта на основе нормалей поверхности.
- Реализация теней путём испускания теневых лучей из точки пересечения к источнику света.
- Многопоточный рендеринг с использованием `Parallel.For` для распределения расчёта пикселей по всем ядрам процессора (в будущем планируется переход на вычисления с помощью GPU и Vulkan).
- Преобразование яркости в символьный градиент `" .:!/r(l1Z4H9W8$@"` для вывода в консоль.
- Буферизация кадров и вывод группами одинаковых цветов для минимизации медленных вызовов системного API консоли.
- Собственные структуры `Vector3`, `Vector2`, `Ray` и матрицы поворота.
- Реализация алгоритма Мёллера — Тримбора для нахождения пересечения луча с треугольником.
- Input-система для эффективного опроса клавиатуры с использованием системного API (подстраивается под конкретную систему за счёт `IInputProvider`). Поддерживаются: `User32.dll` на Windows, `X11` (с отключённым Wayland) на Linux, а также `DotNet Input` (неэффективный вариант с ограничениями по количеству одновременных нажатий и сложностью отслеживания клавиш-модификаторов `Shift`, `Ctrl`, `Alt`; используется в случаях, когда в системе нет доступа к `User32` или `X11`).
- Самописный клиент-серверный модуль на основе TCP-архитектуры для передачи игровых данных. В будущем планируется добавление стабильного UDP-интерфейса для увеличения скорости в онлайн-сценах.
- Кастомная бинарная сериализация пакетов, автоматическая маршрутизация через стабильные хэш-идентификаторы типов и система подписок на события.
- Примеры работающего мультиплеер-лобби с синхронизацией позиций игроков и текстовым чатом.
- Пример работающей сцены с импортом `.obj`-объекта.
- Автоматическое считывание размера консоли и отрисовка под её максимальный размер (определяется при запуске).

## Условия эффективного запуска

При запуске на Linux в терминале должен быть отключён Wayland (Wayland не предоставляет терминалу прямого доступа к X11). Способов отключения много. Если Wayland активен, при запуске сцены отобразится предупреждение.

Самый простой способ запустить терминал без него — использовать независимый эмулятор терминала (например, `xterm`), принудительно запущенный в режиме совместимости с X11 (через XWayland):
```bash
apt install xterm
WAYLAND_DISPLAY= xterm
```

В открывшемся окне `xterm` перейдите в папку вашего проекта и запустите сборку.


## Сборка

Neo 3D не использует сторонние библиотеки. Для запуска нужен лишь **.NET 8.0 SDK** или новее.
1. Скачайте или клонируйте проект с GitHub.
2. Перейдите в папку `SampleGame` (внутри проекта) через терминал и скомпилируйте проект: `dotnet run --configuration Release`
3. Если вы всё сделали правильно, появившейся сценой можно будет управлять. Используйте `W`, `A`, `S`, `D` для перемещения; `Space`, `Shift` (или `Shift + любая клавиша` в случае с `dotnet input`) — для управления высотой; стрелки — для поворота камеры. Клавиши `+` и `-` служат для повышения и понижения яркости света на сцене. На сцене с 3D-объектом при зажатой клавише `Ctrl` вы можете управлять источником света (перемещать его).

## Пример создания собственной сцены (C#) (взаимодействие с API движка)

```csharp
using _3dEngine;
using _3dEngine.AbstractClass;
using _3dEngine.Implementation;
using _3dEngine.Interfaces;
using _3dEngine.Shape;

public class PreviewScene : Scene
{
    private Camera _camera;
    private Sphere _sphere;
    private Light _light;

    public PreviewScene(IDisplaysManagerAsync manager) : base(manager) { }

    public override void Start()
    {
        // Создаем камеру
        _camera = new Camera(new Vector3(-10, 0, 0), Vector3.Zero);
        SetMainCamera(_camera);

        // Создаем красную сферу в центре координат
        _sphere = new Sphere(Vector3.Zero, Vector3.Zero, r: 1.5f);
        _sphere.Color = ConsoleColor.Red;
        AddDisplaysObject(_sphere);

        // Добавляем источник света
        _light = new Light(new Vector3(-4, 3, -2), lightPower: 150f);
        AddLight(_light);
    }

    public override void Update()
    {
        // Вращаем или перемещаем объекты каждый кадр, используя GameTime.GetDeltaTime()
    }
}
```

Для запуска сцены её надо указать в запускаемом файле `Program.cs` при создании `Frame`:

```csharp
using _3dEngine;
using _3dEngine.Implementation;

class Program
{
    static void Main()
    {
        new Frame(new MyCustomScene(new DisplayManagerAsync()), new ConsoleScreenAsync()).MainLoop();
    }
}
```
