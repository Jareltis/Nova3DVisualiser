# Nova3DVisualiser

3D-движок реального времени, который рендерит изображение в терминал в виде ASCII-арта, написанный на C# / .NET 8. Это CPU-рейтрейсер: каждый кадр он трассирует лучи по сцене, отображает яркость в градиент символов и (на терминалах с truecolor) подкрашивает каждую ячейку 24-битным RGB-цветом, рисуя результат прямо в консоли.

В комплекте идёт мультиплеерное демо-приложение с полноценным **встроенным редактором сцены**, **системой миров** (каждая сцена — это один JSON-файл, который можно создать, сохранить и загрузить заново), **богатым цветным освещением** и базовой TCP-сетью с чатом.

---

## Возможности

### Рендеринг
- Трассировка лучей на CPU с выводом в ASCII через градиент яркости.
- **Truecolor-вывод**: каждая ячейка несёт 24-битный RGB-цвет (`Rgb24`), выводимый ANSI-кодами, поэтому картинка больше не ограничена 16 цветами консоли. На терминалах без truecolor используется ближайший консольный цвет.
- Конвейер освещения с плавающей точкой, поканальной **тональной компрессией по Рейнхарду (Reinhard)**, фоновой (ambient) составляющей и регулируемой экспозицией.
- Сглаженное (по нормалям вершин) затенение, с дружелюбным к Blender загрузчиком OBJ (триангуляция n-угольников веером, отрицательные/относительные индексы, запасной вариант с геометрической нормалью).
- **BVH-ускорение** для высокополигональных мешей, строится один раз в локальном пространстве и обходится с лучом, преобразуемым каждый кадр, поэтому работает даже с анимированными/вращающимися моделями.
- Отсечение по ограничивающей сфере для каждого объекта — как для первичных лучей, так и для теневых.
- Опциональные тени (переключаются для каждого мира).
- Регулируемая детализация рендера (1–4): меняйте разрешение на скорость прямо в процессе одной клавишей.

### Освещение
В Nova реализована полноценная система множественного освещения. Каждый источник света проверяется на тень **независимо** и вносит вклад аддитивно, поэтому поверхность, затенённая от одного источника, всё равно освещается другим (нет единственной доминирующей тени). Виды источников:

- **Point (точечный)** — всенаправленный, с обратно-квадратичным затуханием.
- **Directional (направленный)** — параллельное «солнце» с постоянным мягким затуханием.
- **Spot (прожектор)** — конус, направленный вдоль вектора, с дополнительным формированием (см. ниже).
- **Area (площадной)** — квадратный излучатель, сэмплируемый по сетке, для площадного затухания и мягких теней.

Дополнительно для каждого источника:

- **Цветное освещение** — каждый источник излучает RGB-цвет (берётся из его `ConsoleColor`). Параметр `SurfaceTint` позволяет цветному свету проявляться на любой поверхности, а не только на поверхности того же цвета.
- **Spin (вращение)** — со временем поворачивает направление источника (рад/с) для анимированного освещения.
- **Многонаправленные лучи прожектора** — прожектор может бросать сразу **1–8 конусов** (`Beams`), равномерно развёрнутых вокруг его оси; точка освещается тем лучом, который до неё дотягивается.
- **Форма конуса прожектора** — сечение конуса может быть **кругом (Circle)**, **квадратом (Square)** или **треугольником (Triangle)**.

### Система миров и редактор
- **Мир** — это один файл `worlds/<name>.json`, владеющий всей сценой: переключатели графики, платформа и полный трансформ каждого объекта (позиция/поворот/масштаб/цвет/привязка, а для источников света — вид/направление/конус/лучи/форма и т.д.).
- `models/` — это **чистая библиотека мешей**, только файлы `<name>.obj`. Мир ссылается на меш по имени и задаёт его размещение.
- **Встроенный редактор сцены** (включается клавишей `Tab`) позволяет создавать, перемещать, поворачивать, масштабировать, перекрашивать и удалять объекты вживую, а затем сохранять обратно в JSON.
- Встроенные типы для спавна: `cube`, `sphere`, `cylinder`, `cone`, `pyramid`, `light`, плюс каждый меш из `models/`.
- **Формы платформы**: квадрат, прямоугольник или круг, с настраиваемым размером/цветом.

### Сеть
- Базовый TCP сервер/клиент с общей сценой и встроенным чатом.
- **Сервер — это владелец мира (authority)**: он отправляет полный мир подключающимся клиентам, а затем транслирует дельты правок вживую. Клиенты могут летать и осматривать сцену, но изменять общий мир может только authority.

### Прочее
- Логирование в файл с отметками времени.
- Контроль частоты кадров (ограничение FPS), дельта-время высокого разрешения и клавиша выхода.

## Требования
- .NET 8 SDK.
- Windows (ввод с клавиатуры использует Win32 API, поэтому приложение сейчас работает только под Windows).
- Терминал с поддержкой 24-битного ANSI-цвета для наилучшего вывода (рекомендуется Windows Terminal).

## Сборка и запуск
```bash
dotnet build -c Release
```
Установите `SampleGame` как стартовый проект в Visual Studio или запустите его напрямую:
```bash
dotnet run --project SampleGame -c Release
```

### Настройка при запуске (меню перед рендером)
Перед циклом рендеринга запускается небольшой мастер на Terminal.Gui:
1. **Режим сессии** — локальная одиночная или онлайн.
2. **Сетевая роль** (только онлайн) — поднять **Server** или подключиться как **Client**.
3. **Мир** — **Создать** новый мир или **Загрузить** сохранённый (`worlds/*.json`).
   - **Создание** позволяет задать имя мира, переключатели (**Shadows** — тени, **BVH acceleration** — BVH-ускорение, **Extra fixed light** — дополнительный фиксированный свет, **Disable camera light** — отключить свет камеры, **Include platform** — включить платформу), **форму платформы** (Square / Rectangle / Circle) и её размер/ширину/глубину.
4. **Сеть** (только онлайн) — порт прослушивания (сервер) или IP сервера + порт (клиент).

Клиент, подключившийся к серверу, скачивает мир хоста автоматически.

## Управление

Клавиши сгруппированы по режимам. Управление камерой (полёт) работает всегда; управление редактором действует только при открытом редакторе (`Tab`); чат перехватывает весь ввод, пока он открыт (`T`).

### Камера / полёт (активно всегда)
| Клавиша | Действие |
| --- | --- |
| **W** / **S** | Движение вперёд / назад (относительно направления взгляда) |
| **A** / **D** | Боковое смещение влево / вправо |
| **Space** / **C** | Движение вверх / вниз |
| **← / →** | Поворот взгляда влево / вправо (рыскание) |
| **↑ / ↓** | Поворот взгляда вверх / вниз (тангаж) |
| **Q** / **E** | Крен камеры влево / вправо |
| **R** | Сбросить крен в горизонт |
| **Z** / **X** | Приблизить / отдалить (FOV уже / шире, 20°–120°) |
| **V** | Сбросить FOV к значению по умолчанию |
| **=** / **-** | Увеличить / уменьшить мощность света камеры |
| **P** | Циклически менять детализацию рендера 1 → 2 → 3 → 4 → 1 (меньше = быстрее) |
| **T** | Открыть чат (в онлайн-сессиях) |
| **Esc** | Выйти из приложения |

### Редактор (нажмите **Tab**, чтобы включить/выключить редактор)
Редактор показывает прицел по центру и панель свойств выбранного объекта. Управление полётом продолжает работать во время редактирования.

| Клавиша | Действие |
| --- | --- |
| **Tab** | Включить/выключить редактор |
| **G** | Переключить тип для спавна (cube / sphere / cylinder / cone / pyramid / light / любой меш из `models/`) |
| **Enter** | Заспавнить текущий тип перед камерой *(только authority)* |
| **F** | Выбор прицелом: взять объект под прицелом |
| **[** / **]** | Выбрать предыдущий / следующий объект |
| **,** / **.** | Перемещать курсор поля вверх / вниз в панели свойств |
| **N** / **M** | Уменьшить / увеличить значение выделенного поля *(только authority)* |
| **I** / **K** | Двигать выбранный объект по +Z / −Z *(только authority)* |
| **J** / **L** | Двигать выбранный объект по −X / +X *(только authority)* |
| **U** / **O** | Двигать выбранный объект вверх / вниз (+Y / −Y) *(только authority)* |
| **Delete** | Удалить выбранный объект *(только authority)* |
| **F5** | Сохранить мир обратно в `worlds/<name>.json` *(только authority)* |

> В онлайн-сессии спавнить/двигать/редактировать/удалять/сохранять может только **authority** (сервер или локальная одиночная сессия). Подключённый клиент может открыть редактор, чтобы **выбирать и осматривать** объекты, но его изменения не затрагивают общий мир.

### Чат (нажмите **T**, чтобы открыть; только онлайн)
| Клавиша | Действие |
| --- | --- |
| *ввод* | Набрать сообщение |
| **Enter** | Отправить |
| **Backspace** | Удалить символ |
| **Esc** | Отменить без отправки |

### Панель свойств (редактируется клавишами `,` `.` `N` `M`)
Список редактируемых полей зависит от типа выбранного объекта:

- **Меш / cube / cylinder / cone / pyramid** — Pos X/Y/Z, Rot X/Y/Z, Scale (масштаб), Spin (скорость авто-вращения), Color (цвет).
- **Sphere** — Pos X/Y/Z, Radius (радиус), Color.
- **Light (источник света)** — Pos X/Y/Z, Power (мощность), Color, **Kind** (вид), затем в зависимости от вида:
  - *Point*: ничего дополнительно.
  - *Directional*: Dir X/Y/Z (направление), Spin.
  - *Spot*: Dir X/Y/Z, Cone (половинный угол), **Beams** (1–8 развёрнутых конусов), **Shape** (Circle / Square / Triangle), Spin.
  - *Area*: Dir X/Y/Z, Size (половинный размер), Spin.

`Color` циклически перебирает 16 именованных консольных цветов; для источника света он также задаёт излучаемый цвет.

## Добавление моделей
Положите меш в `SampleGame/models/` как `<name>.obj` (экспортируйте с нормалями вершин для сглаженного затенения; Blender делает это по умолчанию). Модели — это чистая библиотека геометрии, **отдельного JSON на модель больше нет**; размещение (позиция, поворот, масштаб, цвет, привязка, вращение) хранится в мире.

Чтобы разместить модель в сцене, либо:
- Заспавните её вживую в редакторе (`G`, чтобы найти её в списке спавна, затем `Enter`), расставьте и нажмите `F5`, чтобы сохранить; либо
- Добавьте запись объекта в `worlds/<name>.json` вручную.

Объект мира выглядит так:
```json
{
  "id": 0,
  "type": "mesh",
  "mesh": "monkey",
  "position": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "rotation": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "scale": 1.0,
  "color": "Yellow",
  "anchor": "bottom",
  "rotateSpeed": 1.0
}
```
- `type` — `mesh`, `cube`, `sphere`, `cylinder`, `cone`, `pyramid` или `light`.
- `mesh` — имя файла в `models/` (без `.obj`), для `type: "mesh"`.
- `position` — позиция в мире в единицах (Y — вертикальная ось).
- `rotation` — начальный поворот в радианах.
- `scale` — равномерный масштаб.
- `color` — имя .NET `ConsoleColor` (например, `Red`, `Cyan`, `Yellow`).
- `anchor` — как меш располагается в своей позиции: `bottom` (основание на полу, по умолчанию), `center` (геометрический центр) или `origin` (исходное начало координат OBJ).
- `rotateSpeed` — автоматическое вращение вокруг вертикальной оси, в радианах в секунду (0 = статично).

Объект `light` дополнительно использует: `power`, `lightKind` (`point`/`directional`/`spot`/`area`), `direction`, `coneAngle`, `beamCount`, `coneShape` (`circle`/`square`/`triangle`), `lightSize` и `lightSpin`.

Отредактируйте модель или JSON мира и просто перезапустите — пересборка не требуется.

## Структура проекта
- `Nova3DVisualiser/` — движок (библиотека): векторная математика, фигуры, трассировка лучей, конвейер цветного освещения, truecolor-вывод в консоль, BVH, загрузка OBJ/мешей, сеть, логирование. Не зависит от сторонних пакетов.
- `SampleGame/` — демонстрационное приложение: мастер настройки, система миров (`worlds/`), встроенный редактор сцены, сетевой хост и библиотека мешей `models/`.
