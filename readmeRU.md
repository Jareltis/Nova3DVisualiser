# Nova3DVisualiser

3D-движок реального времени, который рендерит изображение в терминал в виде ASCII-арта, написанный на C# / .NET 8. Это CPU-рейтрейсер: каждый кадр он трассирует лучи по сцене и отображает яркость в градиент символов, рисуя результат прямо в консоли.

## Возможности
- Трассировка лучей на CPU с выводом в ASCII через градиент яркости.
- Конвейер освещения с плавающей точкой: несколько суммируемых точечных источников света, фоновая (ambient) составляющая и тональная компрессия по Рейнхарду (Reinhard).
- Сглаженное (по нормалям вершин) затенение, с дружелюбным к Blender загрузчиком OBJ (триангуляция n-угольников веером, отрицательные/относительные индексы, запасной вариант с геометрической нормалью).
- Загрузка моделей из папки: положите `<name>.obj` + `<name>.json` в `models/` и настройте позицию, поворот, масштаб, цвет, вращение и привязку (anchor) — пересборка не нужна.
- BVH-ускорение для высокополигональных мешей, строится один раз в локальном пространстве и обходится с лучом, преобразуемым каждый кадр, поэтому работает даже с анимированными/вращающимися моделями.
- Отсечение по ограничивающей сфере для каждого объекта — как для первичных лучей, так и для теневых.
- Опциональные тени (переключаются при запуске).
- Базовая TCP-сеть (сервер/клиент) с общей сценой и встроенным чатом.
- Логирование в файл с отметками времени.
- Контроль частоты кадров (ограничение FPS) и клавиша выхода.

## Требования
- .NET 8 SDK.
- Windows (ввод с клавиатуры использует Win32 API, поэтому приложение сейчас работает только под Windows).

## Сборка и запуск
```bash
dotnet build -c Release
```
Установите `SampleGame` как стартовый проект в Visual Studio или запустите его напрямую:
```bash
dotnet run --project SampleGame -c Release
```
При запуске вы выбираете Server или Client, IP и порт, а также несколько переключателей (дополнительный свет, отключить собственный свет, тени, BVH).

## Управление
- **W A S D** — перемещение по горизонтали (относительно того, куда вы смотрите).
- **Space / C** — перемещение вверх / вниз.
- **Стрелки** — осмотреться.
- **T** — открыть чат (сетевой режим).
- **Escape** — выход.

## Добавление моделей
Положите меш и его конфигурацию рядом друг с другом в `SampleGame/models/`:
- `dragon.obj` — геометрия (экспортируйте с нормалями вершин для сглаженного затенения; Blender делает это по умолчанию).
- `dragon.json` — его конфигурация:
```json
{
  "name": "Dragon",
  "position": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "rotation": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "scale": 1.0,
  "color": "Yellow",
  "rotateSpeed": 1.0,
  "anchor": "bottom"
}
```
- `position` — позиция в мире в единицах (Y — вертикальная ось).
- `rotation` — начальный поворот в радианах.
- `scale` — равномерный масштаб.
- `color` — имя .NET `ConsoleColor` (например, `Red`, `Cyan`, `Yellow`).
- `rotateSpeed` — автоматическое вращение вокруг вертикальной оси, в радианах в секунду (0 = статично).
- `anchor` — как меш располагается в своей позиции: `bottom` (основание на полу, по умолчанию), `center` (геометрический центр) или `origin` (исходное начало координат OBJ).

Отредактируйте модель или её JSON и просто перезапустите — пересборка не требуется.

## Структура проекта
- `Nova3DVisualiser/` — движок (библиотека): векторная математика, фигуры, трассировка лучей, освещение, BVH, загрузка OBJ/моделей, сеть, логирование.
- `SampleGame/` — демонстрационное приложение и сетевой хост, а также папка ресурсов `models/`.
