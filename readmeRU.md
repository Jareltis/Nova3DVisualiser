# Nova3DVisualiser

*(In English — [readme.md](readme.md))*

Мультиплеерный **ASCII 3D-рейтрейсер**, рендерящий прямо в терминал, написанный на C# / **.NET 10**. По умолчанию работает на многопоточном CPU-рендерере, с опциональным путём на **GPU** (ILGPU / CUDA) при попиксельном паритете. В комплекте — полноценный встроенный редактор сцены, система миров на JSON, настоящая физика твёрдых тел, PNG-текстуры и TCP + UDP мультиплеер, а весь его UI, включая мастер настройки, работает на собственном консольном рендерере движка, **без внешнего UI-тулкита**.

Проект развивается в сторону движка, чей **встроенный редактор сам является игрой** — редактор-первой песочницы, где создаваемые вами миры, модели и текстуры — **общий локальный контент**.

## Возможности

- **Рендерер CPU или GPU, для каждого мира** — многопоточный CPU-рейтрейсер по умолчанию либо путь ILGPU/CUDA при полном паритете фич (прозрачный откат, если GPU нет).
- **Truecolor ASCII** — 24-битный RGB на ячейку через ANSI, с diff-рендерингом (перезаписываются только изменившиеся ячейки).
- **Система множественного света** — point / directional / spot / area, цветное излучение, вращение, многолучевые прожекторы, конусы круг/квадрат/треугольник, альфа-корректные тени.
- **Две прозрачности** — альфа объекта (сквозная, front-to-back) и блёклость цвета (выцветание к белому).
- **PNG-текстуры + UV** — все примитивы и импортированные меши; тайлинг, выбор грани и фильтрация nearest/bilinear/mipmapped; пиксели передаются пирам.
- **Настоящая физика твёрдых тел** — импульсный решатель: гравитация, пер-объектная коллизия (AABB/OBB), трение, трение качения, упругость, масса, 3D-кувыркание и устойчивые стопки; **динамические выпуклые оболочки** — произвольный меш падает, кувыркается и стопкуется как его настоящая форма (и лежит на реальных треугольниках наклонных мешей); **шарниры** — ball-socket, hinge (с лимитами угла и мотором) и distance (жёсткий или пружина) — авторятся в редакторе и синхронизируются в мультиплеере, причём шарнир можно прикрепить к **любому** объекту (нефизическая сторона — свет, камера, статичный объект или платформа — работает как фиксированный якорь, который можно двигать в редакторе, перемещая то, что на нём висит), оба тела шарнира могут по желанию **сталкиваться друг с другом** (настоящая цепь упирается, а не проходит насквозь), а редактор показывает, почему шарнир неактивен; и **капсульный** персонаж от первого лица с реальным ростом тела.
- **Система миров + встроенный редактор** — каждая сцена — это один `worlds/*.json`; спавн, перемещение, правка и сохранение объектов вживую.
- **Камеры и HUD** — виды от 1-го/2-го/3-го лица, размещаемые Fixed/Follow камеры, разделённый экран и три режима HUD (play / overlay / docked).
- **Мультиплеер** — TCP + UDP сервер/клиент: надёжный TCP для синхронизации мира (чанкованные меши), правок, спавнов и чата; ненадёжный UDP fast-path (latest-wins, MTU-чанкинг, откат на TCP) для трансформов игроков и синхронизации физики. Сообщения в чате теперь помечаются **никнеймом** отправителя, а список никнеймов (roster) рассылается всем пирам (используется в чате сейчас, а дальше — для подписей при наведении).

## Быстрый старт

**Требования:** .NET 10 SDK · Windows · терминал с 24-битным цветом (рекомендуется Windows Terminal) · *(опционально)* GPU NVIDIA/CUDA.

```bash
dotnet build -c Release
dotnet run --project SampleGame -c Release
```

Дальше всё берёт на себя брендированный **мастер настройки** — никнейм, режим сессии, сетевая роль, мир (создать/загрузить) и сетевые параметры. Он начинается с вопроса, кто играет: выберите ранее использованный никнейм или введите новый. Никнеймы запоминаются локально в `SampleGame/users.json` (в `.gitignore`) и не ограничивают и не изолируют контент — миры, модели и текстуры остаются общими локальными файлами независимо от никнейма.

### Тестовая лаборатория (Test Lab)

Сгенерируйте готовый мир **«TestLab»**, чтобы проверять физику по заведомо правильным примерам:

```bash
dotnet run --project SampleGame -c Release -- maketestworld
```

Затем запустите игру и **загрузите мир «TestLab»**. В нём три полосы:

- **Зона J** (витрина шарниров) — маятник ball-socket, жёсткий стержень, пружина, дверь на петле с лимитами ±90°, мельница с мотором (вращается перед своим столбом), цепь из 3 звеньев, чьи звенья **сталкиваются как настоящая цепь** (упираются друг в друга, а не проходят насквозь), качели на двух стержнях и **люк**, который при загрузке распахивается и повисает на лимите петли (лимиты видны без всякого ввода). Пересоздать в любой момент можно командой `maketestworld` — она обновит существующий TestLab.
- **Зона P** (физическая площадка) — стопка кубов, прыгучая сфера, рампа со скатывающейся сферой и пара тяжёлый/лёгкий для толчков.
- **Зона C** (капсульная полоса) — перекладина на уровне головы, под которую не пролезть, слалом и низкая ступенька, на которую можно зайти.

## Участие

Nova3DVisualiser пишется исключительно его мейнтейнером и **не принимает вклад в код** — пул-реквесты закрываются без слияния. При этом **отчёты об ошибках, предложения функций и идеи очень приветствуются** через [GitHub Issues](https://github.com/Jareltis/Nova3DVisualiser/issues).
